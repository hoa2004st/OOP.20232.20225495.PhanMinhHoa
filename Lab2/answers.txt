Q1: If you create a constructor method to build a DVD by title then create a constructor method to build a DVD by category. Does JAVA allow you to do this?
Answer: No, that is because they have the same signature (including methode name, quantity and data type of parameter)

Q2: Is JAVA a Pass by Value or a Pass by Reference programming language?
Answer: It is a Pass by Value programming language. 
When passing a primative data type, it copy the value.
When passing a reference data type, it copy the reference, not the object.

Q3: After the call of swap(jungleDVD, cinderellaDVD) why does the title of these two 
objects still remain?
Answer: That is because the assign function (o1 = o2 ,...) change the object locally, not globally

Q4:  After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()) why is 
the title of the JungleDVD changed?
Answer: That is because the attribute of the dvd is set to another value.
